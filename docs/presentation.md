# AI 私人医生 —— 边缘智能健康助手演讲报告

> 5 分钟版 · 嵌入式竞赛展示用

---

## 开场（30 秒）

尊敬的评委老师、同学们：

大家好！我是 XXX，今天为大家带来我们团队的作品——**AI 私人医生**。这是一款基于 RK3588 边缘计算平台的智能健康助手，集语音问诊、拍照问药、用药提醒、紧急求助于一体，致力于让老年人足不出户就能享受专业的医疗健康服务。

---

## 一、痛点与挑战（45 秒）

我们注意到几个现实问题：

1. **就医不便**：老年人去医院排队挂号难，小病小痛往往拖着；
2. **用药安全**：老人常忘记服药或吃错药，子女不在身边无法及时提醒；
3. **健康管理缺失**：日常健康数据（心率、步数等）缺乏系统化管理；
4. **紧急情况难处理**：突发状况时，老人可能无法及时联系家人；
5. **网络依赖**：传统智能设备依赖云端，断网即失效，稳定性差。

---

## 二、核心创新点（1 分钟）

我们的解决方案有三个核心创新：

**创新一：端云协同架构**

```
  板端（本地）              云端（DashScope）
┌─────────────────┐       ┌─────────────────┐
│ KWS 唤醒 "小医" │       │  Qwen 大模型     │
│ Sherpa ASR      │       │  Qwen-VL 视觉    │
│ Piper TTS       │       │  CosyVoice 语音  │
│ Qwen2-VL RKLLM  │       └─────────────────┘
│ FAISS RAG       │
└─────────────────┘
```

关键设计：**本地模型保证基础功能离线可用，云端模型提供更高质量服务**。环境变量一键切换，互为降级备份。

**创新二：全链路语音交互**

从唤醒到回答，全程语音：

1. **KWS 唤醒**：说"小医"即可激活，带 RMS 能量门控防误触发；
2. **ASR 识别**：板端 Sherpa-ONNX，RTF 仅 0.04，实时转写；
3. **LLM + RAG**：结合疾病/药品知识库，回答更准确；
4. **TTS 播报**：Piper 本地合成，秒级响应。

**创新三：实时拍照问药**

进入拍照界面即启动摄像头 MJPEG 实时预览，语音播报"两秒后拍照"时画面实时显示，让用户有充足时间对准药盒。拍照后由视觉大模型自动识别药名、用法用量、注意事项。

---

## 三、核心功能演示（1 分 30 秒）

### 功能一：语音问诊

- 说"小医，我头疼怎么办？"
- 系统自动唤醒 → ASR 识别 → LLM 分析 → TTS 播报回答
- 支持多轮连续对话

### 功能二：拍照问药

- 点击拍照识药 → 摄像头实时预览 → 语音提示"两秒后拍照"
- 自动识别药盒 → 显示药名、用法用量、注意事项
- 支持"继续问药"了解更多，"换一个"重新拍照

### 功能三：用药提醒

- 每日服药时间到点自动弹窗 + TTS 播报
- 支持"已服用"确认，漏服时重复提醒
- 数据同步到手机端，子女可远程查看

### 功能四：健康大屏

- Apple Health 风格可视化界面
- 展示健康评分、今日用药、待办提醒
- ECharts 趋势图展示心率/步数等数据

### 功能五：SOS 紧急求助

- 一键发送微信消息给家属
- 无冷却期，紧急时刻随时可用
- 消息包含位置信息和健康状态

### 功能六：每日健康日报

- 定时汇总当日健康数据
- 推送到家属微信
- 包含问诊记录、用药情况、健康指标

## 四、边缘 AI 性能（45 秒）

我们对板端模型进行了全面的性能测试：

| 能力 | 延迟 | 准确率 | 资源占用 |
|------|------|--------|----------|
| KWS 唤醒 | 560ms | 漏唤醒率 0% / 误唤醒率 0% | 95MB |
| ASR 识别 | RTF 0.04 | 中文 CER 7.7% | - |
| RAG 检索 | 68ms | Top3 命中率 90% | - |
| 本地 TTS (Piper) | 1.06s | RTF 0.42 | 944MB |
| 系统峰值 | - | - | CPU 70.8% / 内存 6GB / 温度 84°C |

关键优势：
- **低延迟**：唤醒 + ASR 全程亚秒级响应，RTF < 0.05
- **离线可用**：本地模型保证基础功能不依赖网络
- **资源可控**：RK3588 NPU 加速，温度稳定在 84°C 以内

> **注**：KWS 唤醒准确率统计口径说明——测试音频集包含非唤醒词样本，漏唤醒率与误唤醒率均为 0%，表明唤醒功能稳定可靠。
